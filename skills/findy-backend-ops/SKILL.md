# FINDY Backend Operations

운영 데이터와 인프라 경로를 작은 단계로 정비한다. 상세 기준은 [운영 계약](references/operations-contract.md)을 따른다.

## 작업 순서

1. 저장소 규칙, 현재 환경 구분, 데이터 마이그레이션과 배포 파일을 읽는다.
2. 기존 로컬 개발 경로를 보존하고 production 동작만 명시적으로 강화한다.
3. 스키마 변경은 버전형 마이그레이션으로 추가하고 롤백·백업 절차를 먼저 기록한다.
4. 중복 지급, 신고 누적, 미디어 상태 같은 경쟁 구간은 DB 제약과 트랜잭션으로 보호한다.
5. health와 readiness를 분리하고 의존 서비스 준비 상태를 검증한다.
6. 로그에서 인증값과 개인정보를 마스킹하고 요청 ID와 핵심 지표를 중앙 수집 경로로 보낸다.
7. 단위 테스트 뒤 가능한 범위에서 격리된 PostgreSQL·Redis·S3 호환 통합 테스트를 실행한다.

## 경계

- 운영에서 seed 데이터를 자동 실행하지 않는다.
- 실제 비밀값, 공급자 URL, 서명 인증서를 저장소에 넣지 않는다.
- SQLite 파일을 파괴하거나 기존 데이터를 무단 변환하지 않는다.
- 외부 서비스가 없는 CI에서는 해당 검증을 명시적으로 skip하고 성공으로 위장하지 않는다.
