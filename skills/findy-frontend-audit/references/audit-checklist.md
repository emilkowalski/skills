# FINDY Frontend Audit Checklist

## 핵심 흐름

- 오프닝 → 로그인 → 가입 → PASS/SMS → 약관 → 홈
- 홈 → 검색/분야 → 결과 → 상세 → 좋아요/저장
- 커뮤니티 → 필터 → 상세 → 댓글/대댓글 → 신고
- 커뮤니티/스냅/비디오 → 작성 → 첨부 → 제출 → 결과
- 내정보 → 프로필 편집 → 나의 FINDY → 저장/좋아요/내 글
- 설정 → 알림/위치/마케팅 → 정책/지원 → 완료 피드백

## 상태 품질

- 로딩 중 레이아웃 크기가 안정적인가?
- 데이터가 없을 때 원인과 다음 행동이 보이는가?
- 오프라인, 시간 초과, 5xx, 401, 409에 맞는 복구 행동이 있는가?
- 제출 성공 전에 완료로 표시하지 않는가?
- 중복 탭이 중복 생성·반응을 만들지 않는가?
- 재시도 후 입력값과 첨부가 유지되는가?

## 레이아웃

- 360x800과 390x844에서 탭·필터·버튼·긴 한국어가 잘리지 않는가?
- 헤더, 하단 내비게이션, FAB, 토스트, 키보드가 서로 겹치지 않는가?
- 마지막 목록 항목과 푸터까지 완전히 스크롤 가능한가?
- 빈 이미지와 오류 이미지도 안정된 비율을 유지하는가?
- 탭 전환과 뒤로가기에서 스크롤 위치가 예측 가능하게 복원되는가?

## 접근성

- 핵심 제어의 터치 영역이 44pt 이상인가?
- tooltip 또는 접근 가능한 이름이 있는가?
- 색만으로 선택·성공·오류를 구분하지 않는가?
- 글자 확대 시 겹침 없이 줄바꿈되는가?
- 키보드 포커스 순서와 제출 동작이 자연스러운가?
- 지원하지 않는 다크모드에서도 강제 라이트 테마가 깨지지 않는가?

## 콘텐츠와 신뢰

- 임시 아이콘, 검은 미디어, 반복 테스트 이미지가 공개 화면에 없는가?
- 테스트 검색어와 개발용 계정 정보가 초기 상태에 남지 않는가?
- 인증 배지가 실제 상태와 일치하고 의미 색이 일관적인가?
- 중복 카테고리, 과장 문구, 준비되지 않은 기능이 노출되지 않는가?
- 버튼 문구가 실행 결과를 구체적으로 설명하는가?
