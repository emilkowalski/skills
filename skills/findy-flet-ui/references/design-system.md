# FINDY Design System Contract

## 제품 방향

FINDY는 스타일을 발견하고, 맥락을 이해하고, 질문하고, 저장·비교해 결정하는 뷰티 커뮤니티다. 화면은 조용하고 간결해야 하며 실제 콘텐츠와 신뢰 정보가 장식보다 먼저 보여야 한다.

## 코드상 기준값

정확한 현재 값은 FINDY 저장소의 `app/components/layout.py`와 `app/ui/tokens.py`를 source of truth로 삼는다.

| 역할 | 현재 값 |
| --- | --- |
| 브랜드 브라운 | `#8B6B4F` |
| 라인·테두리 | `#E6D7C8` |
| 배경·카드 | `#FFFFFF` |
| 강한 텍스트 | `#1F1A17` |
| 본문 텍스트 | `#2B2420` |
| 보조 텍스트 | `#786A62` |
| 최소 UI 글자 | `12` |
| 본문 글자 | `14` |
| 최소 터치 영역 | `44` |
| 기본 버튼 높이 | `44` |

브랜드 로고와 심볼은 `assets/logo`의 파일을 사용한다. 글자로 로고를 재현하거나 사용자가 확정한 에셋을 임의로 다시 그리지 않는다.

## 의미 색

일반 탐색은 브라운·검정·회색을 사용한다. 색은 신뢰와 상태를 구분할 때만 쓴다.

| 상태 | 전경 | 연한 배경 |
| --- | --- | --- |
| 방문 인증 | `#4E8B66` | `#EEF6F1` |
| 경험 인증 | `#4F7FA8` | `#EEF4F8` |
| 아티스트 답변 | `#8067B2` | `#F3EFFA` |
| 운영 검토·정보 | `#8B6B4F` | `#F4EEE8` |
| 오류 | `#B85C5C` | `#FBEFEF` |
| 경고 | `#D97757` | `#FFF1EC` |

## 공통 컴포넌트

- 카드 안에 카드를 중첩하지 않는다.
- 텍스트 칩은 `findy_text_chip`을 우선 사용한다.
- 뒤로가기는 `findy_back_button`을 사용한다.
- 상단 액션은 `findy_header_action`으로 크기와 검정 glyph를 통일한다.
- 토스트는 `present_findy_toast`를 사용하고 하단 내비게이션 위 여백을 계산한다.
- 고정 헤더와 하단 내비게이션은 콘텐츠 위에 겹쳐 보이지 않아야 한다.
- 선택된 좋아요·저장은 아이콘 채움으로만 표현하고 배경을 바꾸지 않는다.

## 콘텐츠 위계

- 헤더: FINDY 로고, 현재 화면명, 필요한 액션만 둔다.
- 피드 카드: 프로필과 신뢰 상태, 제목, 본문 미리보기, 태그, 미디어, 반응 순서로 읽히게 한다.
- 검색 결과 메타는 `분야 · 콘텐츠 유형` 순서로 표시한다.
- 같은 정보나 카테고리를 카드 안에서 두 번 반복하지 않는다.
- 설명 문구는 행동 판단에 필요할 때만 남긴다.

## 반응형과 안전영역

- 기준 뷰포트는 `360x800`, `390x844`를 포함한다.
- 44pt 터치 영역은 시각 크기보다 커도 되지만 주변 요소와 겹치면 안 된다.
- 긴 탭은 숨겨진 채 잘리지 않아야 하며 마지막 항목까지 이동 가능해야 한다.
- 하단 콘텐츠 여백은 내비게이션, FAB, 토스트, 키보드 상태를 함께 고려한다.
- 큰 글자에서 버튼·탭의 글자를 축소해 끼워 넣지 않는다.
