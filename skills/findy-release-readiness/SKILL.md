# FINDY Release Readiness

릴리스 상태를 코드와 증빙으로만 판정한다. 상세 기준은 [릴리스 게이트](references/release-gates.md)를 따른다.

## 작업 순서

1. 저장소 규칙과 릴리스 설정의 source of truth를 확인한다.
2. 제품명, bundle/application ID, 버전, 빌드 번호가 앱·CI·스토어 문서에서 일치하는지 검사한다.
3. 공개 정책·지원·계정 삭제 URL은 HTTPS 접근성과 콘텐츠를 함께 검증한다.
4. Android/iOS release signing 사전검사와 debug 산출물 제출 차단을 확인한다.
5. 운영자 정보, 연령 정책, 데이터 공개, APNs/FCM, 공급자 설정의 미확정 항목을 blocker로 유지한다.
6. 저장소의 릴리스 검사, 스토어 검사, 배포 계약 검사와 전체 테스트를 실행한다.

## 경계

- 임시 URL, 가짜 연락처, 더미 키로 ready 상태를 만들지 않는다.
- 법률 문구와 스토어 답변을 임의 확정하지 않는다.
- 시뮬레이터와 자동 검사로 실기기·실서명 통과를 선언하지 않는다.
- 외부 입력이 필요한 항목은 담당자, 필요한 값, 확인 방법을 명확히 남긴다.
