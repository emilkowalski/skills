# FINDY API Security

FINDY 인증과 사용자 API의 신뢰 경계를 보존하는 구현 스킬이다. 상세 계약은 [보안 계약](references/security-contract.md)을 따른다.

## 작업 순서

1. 저장소 규칙과 `git status --short`를 확인하고 기존 변경을 보존한다.
2. `auth/`, `api/`, `security/`, `tests/`의 현재 인터페이스와 마이그레이션 상태를 읽는다.
3. 인증된 사용자 ID는 검증된 세션 또는 서명 토큰에서만 결정한다. 요청 본문의 사용자 ID를 권한 근거로 사용하지 않는다.
4. OTP·OAuth·PASS state는 repository 인터페이스 뒤에 두고 운영에서는 TLS 공유 저장소 없이는 시작하지 않는다.
5. 소유권, 역할, 재사용 방지, 만료, 시도 제한을 트랜잭션 또는 원자 연산으로 구현한다.
6. 정상 경로와 함께 무인증, 타인 리소스, 만료·재사용 토큰, 동시 요청, 저장소 장애 테스트를 추가한다.
7. 비밀값이나 운영 URL을 코드·테스트 fixture·문서에 넣지 않는다.
8. 관련 단위·통합 테스트, 스모크, 릴리스 검사와 `git diff --check`를 실행한다.

## 경계

- 관리자 API 키는 서버 간 또는 관리자 전용 클라이언트에만 둔다.
- 개발용 기본 키나 광범위 CORS로 운영 서버가 시작되면 안 된다.
- 인증 실패를 우회하는 fallback을 만들지 않는다.
- 공급자 자격 증명과 법무 결정을 추측하지 않는다.
- 외부 공급자를 실행하지 않은 테스트는 완료로 표시하지 않는다.
