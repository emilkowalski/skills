# FINDY Mobile QA

모바일 화면의 잘림과 상호작용 회귀를 재현하는 QA 스킬이다. 자동화 결과와 실제 기기 결과를 분리하고, 시뮬레이션만으로 실기기 통과를 선언하지 않는다.

## 테스트 준비

1. 저장소 규칙과 `docs/FRONTEND_QA.md`, 모바일 QA 문서가 있으면 읽는다.
2. `git status --short`로 기존 변경을 확인한다.
3. 공식 진입점으로 앱을 실행하고 오류 로그를 보존한다.
4. [뷰포트 매트릭스](references/viewport-matrix.md)에서 요청 범위에 맞는 시나리오를 선택한다.
5. 각 결함에 화면, 크기, 글자 배율, 입력 상태, 재현 절차를 기록한다.

## 우선 재현 항목

- 커뮤니티 탭과 필터의 마지막 항목 잘림 또는 서로 압축되는 현상
- 스냅 분야 탭, 정렬, 보기 전환의 좁은 화면 충돌
- 설정 스크롤 중 FINDY 헤더 크롭
- 하단 내비게이션, FAB, 토스트, 마지막 콘텐츠의 겹침
- 키보드 등장 시 가입·글쓰기·댓글 CTA 가림
- 뒤로가기 시 화면 이력과 스크롤 위치 미복원
- 비디오 로딩, 소리, 재개, 탭 이탈 정지
- 이미지 실패 시 레이아웃 이동 또는 빈 검은 영역

## 수정 규칙

- 사용자가 수정까지 요청한 경우에만 코드를 고친다.
- 기능 범위를 확장하지 않고 공통 safe-area와 primitive를 우선 수정한다.
- 고정 너비를 추가하기 전에 `build_layout_metrics`, `findy_bottom_clearance`, 공통 sticky tab과 shell을 재사용한다.
- 가로 스크롤은 마지막 항목이 완전히 도달 가능하고 현재 선택이 보일 때만 사용한다.
- 키보드가 열린 동안 핵심 CTA가 스크롤 또는 안전한 고정 영역으로 접근 가능해야 한다.
- 큰 글자에서 텍스트를 임의 축소하지 않는다. 줄바꿈, 컨테이너 높이, 레이아웃 재배치로 해결한다.
- 다크모드를 지원하지 않으면 시스템 설정에서도 라이트 테마와 명암이 깨지지 않는지 확인한다.

## 증빙과 보고

수정 전후에 동일한 조건을 비교한다. 스크린샷은 증빙용으로 만들 수 있지만 사용자가 요청하지 않으면 최종 답변에 표시하지 않는다.

결과에는 다음을 포함한다.

- 통과·실패·미확인 수
- 차단 및 높은 우선순위 결함
- 수정한 파일과 회귀 위험
- 자동 검증 결과
- Android/iOS 실기기에서만 확인 가능한 남은 항목

실기기, VoiceOver, TalkBack, 카메라·위치·알림 권한, APNs/FCM, 실제 네트워크 전환을 실행하지 않았다면 반드시 `미확인`으로 남긴다.
