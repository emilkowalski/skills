# FINDY Repository Maintenance

대규모 정리를 기능 변경과 분리해 추적 가능하게 수행한다. 상세 기준은 [저장소 지도](references/repository-map.md)를 따른다.

## 작업 순서

1. `git status --short`, 추적 파일, ignore 규칙, 저장소 규칙을 읽는다.
2. 실제 진입점과 import·문서·CI·Docker·배포 참조를 전부 검색한다.
3. 이름 변경표와 보관·삭제 후보를 만든다. 사용자가 확정한 에셋과 문서는 임의 변형하지 않는다.
4. 이동은 한 소유권 영역씩 수행하고 모든 참조를 같은 단계에서 갱신한다.
5. 미사용 여부가 불확실하면 삭제하지 않고 `archive/`에 목적과 복구 경로를 기록한다.
6. 중복된 호환 경로는 소비자가 모두 전환된 뒤 제거한다.
7. 컴파일, 단위 테스트, 스모크, 사이트 검사, 공개 저장소 안전성, `git diff --check`를 실행한다.

## 경계

- 사용자의 커밋되지 않은 변경을 되돌리지 않는다.
- `.env`, DB, 키, 개인정보 문서, 민감 PDF는 삭제하지 말고 Git 추적만 해제한다.
- 파일명만 짧게 만들기 위해 표준 역할 경계를 무너뜨리지 않는다.
- 실행 경로가 확인되지 않은 파일을 미사용으로 단정하지 않는다.
- 커밋과 푸시는 사용자가 요청한 경우에만 보안 검사와 승인을 거쳐 수행한다.
